# 3D Panels Store

Интернет-магазин 3D панелей (Гипс, Дерево, Мягкие панели). Современный дизайн, высокая производительность и удобство управления.

## Стек технологий
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Icons**: Lucide React
- **State**: React Context API (Корзина)

## Быстрый старт

1. Установите зависимости:
```bash
npm install
```

2. Запустите режим разработки:
```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) для просмотра.

## Инструкция для разработчика

### 1. Структура проекта
- `src/app`: Роутинг и страницы (каталог, товар, корзина, оформление).
- `src/components`: Повторно используемые UI-компоненты.
- `src/context`: Глобальное состояние (CartContext для корзины).
- `src/lib`: Утилиты (например, `cn` для объединения Tailwind классов).
- `src/types`: Определения интерфейсов TypeScript.
- `public`: Статические файлы (изображения, иконки).

### 2. Управление товарами
На данный момент товары хранятся в массиве `PRODUCTS` внутри файла:
`src/app/catalog/page.tsx`

Чтобы добавить или изменить товар, отредактируйте объект в этом массиве:
```typescript
{
  id: "1",
  name: "Название товара",
  slug: "url-tovara",
  category: "polyurethane", // категория из списка типов
  price: 1850,
  images: ["/images/products/item-01.jpg"], // пути к фото
  description: "Описание товара...",
  specifications: { 
    width: 500, 
    height: 500, 
    depth: 20, 
    material: "Материал" 
  }
}
```

### 3. Работа с фотографиями
1. Загрузите изображения в папку `public/images/products/`.
2. В объекте товара укажите путь, начиная от корня public: `/images/products/your-image.jpg`.
3. Рекомендуемый формат: JPG или WebP, соотношение сторон 4:5 для сетки каталога.

### 4. Дизайн и кастомизация
- **Цвета**: Основные цвета определены через CSS-переменные в `src/app/globals.css` (ищите блок `:root`).
- **Шрифты**: Настроены в `src/app/layout.tsx` (используется стандартный Geist или системные шрифты).
- **Конфигурация Tailwind**: Все расширения темы находятся в `tailwind.config.ts`.

### 5. Процесс оформления заказа
Логика оформления заказа находится в `src/app/checkout/page.tsx`. На данный момент реализован пошаговый интерфейс (просмотр корзины -> ввод данных -> подтверждение). Данные о заказе можно отправлять на API или email в функции завершения заказа.

## Работа с Git и GitHub

### Основные команды

1. **Сохранить изменения локально:**
```bash
git add .
git commit -m "Описание того, что вы изменили"
```

2. **Отправить изменения на GitHub:**
```bash
git push origin main
```

3. **Получить изменения с GitHub (если работаете с другого устройства):**
```bash
git pull origin main
```

### Решение проблем
- **Если GitHub отклоняет push:** Если вы создали файлы на сайте GitHub (например, README), которых нет у вас на ПК, используйте `git pull origin main --allow-unrelated-histories` или принудительно отправьте свой код командой `git push -f origin main` (осторожно, это сотрет историю на GitHub и заменит её вашей).

## Деплой (Показ сайта заказчику)

Сайт настроен для работы с **Vercel**:
1. Подключите ваш GitHub аккаунт к [vercel.com](https://vercel.com).
2. Импортируйте репозиторий `3d-panels`.
3. Vercel автоматически создаст ссылку вида `3d-panels.vercel.app`.
4. Каждое обновление в ветке `main` (после `git push`) будет автоматически обновлять живой сайт.
